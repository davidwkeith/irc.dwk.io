# Brand and design decisions — `irc://`

Generated by Anglesite design-interview (escape-hatch mode — see below).
Last updated: 2026-04-24.

## How this was decided

Standard `/anglesite:design-interview` is a 4-stage conversation (Intent → Mood → Brand anchoring → Axis confirmation). The skill explicitly offers a "Design it for me" escape hatch when the owner has clear opinions and doesn't want to walk through every question one-by-one. We took that path because the project already had its posture nailed down across `~/Developer/github.com/davidwkeith/irc-::/spec/prd.md` and `~/Developer/github.com/davidwkeith/irc-::/CLAUDE.md` — voice (infrastructural / civic / welcome-mat for the product surface; activist register lives on a separate URL), audience (developer-adjacent + civic adopters), brand (the wordmark `irc://` is the protocol scheme itself).

Owner-side input was a single sentence: *"Let's market to Gen-Z from Gen-X, that's the feel."* That collapsed the design space to one specific register: the elders know how this works, the kids see through corporate polish, and the meeting point is craft-over-flash with a terminal/RFC heritage that Gen-Z reads as authentic and Gen-X experiences as native.

## Reference set

Sites that hit the same chord and were used to triangulate the visual register:

- [htmx.org](https://htmx.org) — brutalist-but-readable, mono-heavy, view-source-friendly
- [tantek.com](https://tantek.com) — IndieWeb pioneer's personal site, microformats-first
- [aaronparecki.com](https://aaronparecki.com) — IndieAuth author, document-first layout
- [pluralistic.net](https://pluralistic.net) — Doctorow's blog, 2-column blog-roll register
- [modern.ircdocs.horse](https://modern.ircdocs.horse) — modern IRC spec viewer
- [htmhell.dev](https://htmhell.dev) — opinionated HTML-craft site, hairline borders
- IETF datatracker — the canonical RFC/draft viewer

What's *not* in the reference set: anything Material- or Tailwind-default, anything with a hero block of stock photography, anything with a gradient.

## Decisions

### Axes

| Axis | Position | Reading |
| --- | --- | --- |
| Temperature | 0.40 | Leaning cool. Warmth comes from typography and tone, not color. |
| Weight | 0.50 | Balanced. Spec text needs breathing room; structure needs density. |
| Register | 0.60 | Leaning authoritative through craft, not through marketing-page polish. |
| Time | 0.20 | Strongly classic. RFC + textfiles + IndieWeb heritage all pull this way. |
| Voice | 0.40 | Subtler than bold but with bone structure. Single-color accent. |

### Palette

Single-accent amber. Everything else is near-black on warm cream paper for light mode, warm off-white on warm near-black for dark mode. The amber gestures at CRT phosphor and IRC's 1988 origin without costuming.

| Token | Light | Dark |
| --- | --- | --- |
| `--color-brand` | `#8C4F00` | `#FFB000` |
| `--color-accent` | `#C77A1A` | `#FFC940` |
| `--color-bg` | `#F8F4ED` | `#0F0E0C` |
| `--color-surface` | `#F1ECE0` | `#1A1714` |
| `--color-text` | `#1A1614` | `#F0E8D8` |
| `--color-muted` | `#5C544E` | `#9A9085` |
| `--color-border` | `#D4CCBC` | `#3A342C` |

WCAG AA contrast verified for body and UI text in both modes.

### Typography

Mono everywhere. Same stack for display and body:

```
ui-monospace, "SF Mono", Menlo, Consolas, "Cascadia Code", "Liberation Mono", monospace
```

System fonts only per ADR-0005. No external font loading. Hierarchy via size and spacing within mono — no font change for headers.

If reading-length tests on the spec pages flag a problem, the cheapest pivot is to swap `--font-body` to `ui-serif, "New York", Georgia, "Iowan Old Style", serif` in `src/design/tokens.css`. Don't introduce a sans-serif anywhere — it would push the whole register toward contemporary/corporate and conflict with every other axis position.

### Shape

Zero radius, no shadows. Hairline borders only.

### What this rules out

- No sans-serif anywhere
- No gradient
- No box-shadow
- No `border-radius` beyond hairline
- No "Join the community" CTA-button styling
- No animations beyond `:focus` rings and link hover (color change only)
- No background-color hero block
- No stock-photography hero
- No icon library beyond inline ASCII or simple SVG
- No second accent color
- No Tailwind/Material/Bootstrap defaults

## Page list

Driven by the project's actual surfaces — spec hosting and adoption discovery — not by the personal-site default.

1. `/` — homepage (elevator pitch, spec triangle, principles, status)
2. `/spec/well-known-irc-json/` — canonical hosted spec for the per-community discovery format
3. `/spec/irc-directory-json/` — canonical hosted spec for directories
4. `/spec/irc-trust-list-json/` — canonical hosted spec for trust lists
5. `/.well-known/irc-directory.json` — the project's published default directory (dogfoods the standard)
6. `/adopters/` — civic and institutional users of `irc://` once we have any
7. `/standards/` — IANA registration status, IRCv3 working-group notes
8. `/blog/` (eventually) — project updates

Spec sub-pages are the most urgent because IANA registrations cite their public URLs. Directory page and adopters page can come later.

## How to override

All visible decisions are CSS custom properties in `src/design/tokens.css`. Change the file, save, and the change is global. To regenerate wholesale, run `/anglesite:design-interview` and pick "start fresh" when asked. To nudge axes incrementally, run the same skill and pick "update specific parts."
